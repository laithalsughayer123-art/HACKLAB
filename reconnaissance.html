Phase 1

Reconnaissance

Discover and gather information about your target using these powerful reconnaissance tools. The first phase of ethical hacking involves collecting data to identify potential vulnerabilities.

Understanding Reconnaissance

Reconnaissance is the critical first step in penetration testing where you gather intelligence about your target. This phase helps identify potential attack vectors before any active scanning or exploitation.

Passive Recon

Collects data without direct interaction (Google searches, WHOIS lookups, public social media).

Active Recon

Involves interacting with the target (port scanning, ping sweeps, DNS enumeration).

Featured Tools

Nmap

A powerful network scanning tool used for host discovery, port scanning, service detection, and vulnerability enumeration.

Port Scanning Service Detection Open Source
Run
How to Use Nmap
nmap -sV 192.168.1.1
🔹 Description:

Nmap (Network Mapper) is a widely used tool for scanning IP addresses, ports, and services. It supports OS detection, version scanning, scriptable interactions, and more — making it essential in reconnaissance and vulnerability assessments.

🚀 Common Use Cases:
  • Scan a single host:
    nmap 192.168.1.1
  • Service and version detection:
    nmap -sV 192.168.1.1
  • Scan a subnet:
    nmap 192.168.1.0/24
  • Aggressive scan (includes OS, script, traceroute):
    nmap -A 192.168.1.1
  • Scan specific ports:
    nmap -p 22,80,443 192.168.1.1
📘 Explanation of Flags:
  • -sS — Stealth SYN scan (default for root)
  • -sV — Detect service versions
  • -O — OS detection
  • -A — Aggressive scan (OS + version + scripts + traceroute)
  • -T4 — Faster scan timing template
⚠️ Notes:
  • Nmap is safe for passive recon but can trigger IDS in aggressive mode.
  • Always scan with proper authorization.
  • Use --script for advanced scans (e.g., --script=vuln).
Recon-ng

A full-featured web reconnaissance framework with a modular interface for collecting open-source intelligence (OSINT).

Reconnaissance Modular Open Source
Run
How to Use Recon-ng
recon-ng
🔹 Description:

Recon-ng is a powerful Python-based web reconnaissance framework designed to gather open-source intelligence (OSINT). It features a command-line interface similar to Metasploit and supports a wide variety of modules for domain, IP, credential, and social media reconnaissance.

🚀 Common Use Workflow:
  1. Start Recon-ng:
    recon-ng
  2. Create a workspace:
    workspaces create acme
  3. Add domain target:
    add domains acme.com
  4. Load a module:
    modules load recon/domains-hosts/bing_domain_web
  5. Run the module:
    run
📘 Useful Commands:
  • show modules — List available modules
  • show options — View module options
  • set [option] [value] — Set a required option
  • marketplace search — Find new modules
  • marketplace install [module] — Install modules from the marketplace
⚠️ Notes:
  • You may need API keys for certain modules (e.g., Shodan, VirusTotal).
  • Recon-ng stores data in a local database that you can export.
  • Great for automation and chaining recon tasks.
Maltego

A powerful OSINT and link analysis platform that visually maps relationships between people, domains, IPs, emails, and social profiles.

OSINT Visual Mapping GUI Tool
Run
How to Use Maltego
maltego
🔹 Description:

Maltego is a GUI-based link analysis and OSINT tool that enables analysts to discover relationships between people, companies, domains, infrastructure, and social media accounts using public and third-party data sources.

🚀 Common Use Workflow:
  1. Start Maltego:
    maltego
  2. Log in or register:
    Use a free or commercial Maltego account to access transforms.
  3. Create a new graph:
    Start with an entity (e.g., domain, person, email address)
  4. Run transforms:
    Right-click the entity → choose transforms (e.g., DNS, whois, social, breach data)
  5. Analyze relationships:
    Use the visual map to explore connections and pivot between entities.
📘 Key Features:
  • Over 100 built-in transforms (DNS, social media, OSINT)
  • Integrates with third-party APIs like Shodan, VirusTotal, HaveIBeenPwned
  • Graph-based view of relationships between data
  • Supports collaboration and report generation
⚠️ Notes:
  • You must register for a Maltego Community or Enterprise license to run transforms.
  • Best used for deep reconnaissance, threat intel, and social engineering prep.
  • Some transforms may require additional API keys (e.g., for builtWith, Shodan).

theHarvester

A powerful OSINT tool for gathering emails, usernames, domains, and hostnames from public sources like Google, Bing, and LinkedIn.

OSINT Email Gathering Open Source
Run
How to Use theHarvester
theHarvester -d example.com -b all
🔹 Description:

theHarvester is a powerful command-line OSINT tool that gathers valuable information such as emails, subdomains, employee names, and hostnames from public sources — including search engines, PGP key servers, and social networks.

🚀 Common Use Cases:
  • Basic scan of a domain using all available sources:
    theHarvester -d example.com -b all
  • Gather emails and hosts using Google only:
    theHarvester -d example.com -b google
  • Output results to an HTML file:
    theHarvester -d example.com -b all -f report
📘 Supported Data Sources:
  • google — Google search
  • bing — Microsoft Bing
  • linkedin — Requires API
  • crtsh — Certificate transparency logs
  • pgp — Public PGP key servers
  • all — Use all available sources
⚠️ Notes:
  • No login is required for basic functionality.
  • Some APIs (e.g., LinkedIn, Twitter) require keys to use.
  • Best used early in the recon phase for footprinting and social engineering prep.
Shodan

A search engine for internet-connected devices. Used to find exposed services, IoT systems, vulnerable servers, and more.

Reconnaissance IoT & Infra API Supported
Run
How to Use Shodan
shodan search apache country:"US"
🔹 Description:

Shodan is a specialized search engine that indexes information about devices connected to the internet. It can identify vulnerable services, IoT systems, industrial control systems, webcams, printers, databases, and more.

🚀 Common Use Cases:
  • Search for Apache servers in the US:
    shodan search apache country:"US"
  • Find MySQL services exposed:
    shodan search port:3306
  • Find IPs running specific software:
    shodan search "nginx 1.18"
  • Get full host info by IP:
    shodan host 8.8.8.8
📘 API Integration:
  • shodan init [API_KEY] — Set up your API key
  • shodan search [query] — Run search from CLI
  • shodan host [ip] — Get details on a specific IP
⚠️ Notes:
  • Basic use is free; advanced filters require a paid account.
  • API is rate-limited unless upgraded.
  • Great for identifying exposed services during the recon phase.
  • Use responsibly — some indexed services are real production systems.
Whois

A query tool that retrieves domain registration and ownership information, often used during reconnaissance to analyze targets.

Reconnaissance Domain Info CLI Tool
Run
How to Use Whois
whois example.com
🔹 Description:

Whois is a command-line utility used to gather domain registration data. It reveals information such as registrar, registrant, nameservers, registration/expiration dates, and contact info — valuable for assessing domain history and attack surface.

🚀 Common Use Cases:
  • Lookup domain owner and contact info:
    whois example.com
  • Check domain registration and expiry:
    whois target.org
  • Find authoritative nameservers:
    whois hacklab.net
📘 Example Output Includes:
  • Registrant name (may be redacted for privacy)
  • Registrar name and WHOIS server
  • Creation, update, and expiry dates
  • Name servers
  • DNSSEC status
⚠️ Notes:
  • Some registrars use privacy protection — data may be masked.
  • Whois can also be queried via web tools (e.g., whois.domaintools.com).
  • Install with sudo apt install whois if not available.
DNSenum

A DNS enumeration tool that performs subdomain discovery, brute-forcing, and zone transfer checks to expose potential attack surfaces.

DNS Recon Subdomain Discovery Open Source
Run
How to Use DNSenum
dnsenum example.com
🔹 Description:

DNSenum is a Perl-based tool used to perform DNS enumeration on target domains. It helps identify subdomains, hostnames, mail servers, name servers, and checks for DNS zone transfers.

🚀 Common Use Cases:
  • Basic domain enumeration:
    dnsenum example.com
  • Enable subdomain brute force:
    dnsenum -f wordlist.txt example.com
  • Check for zone transfers:
    dnsenum --enum example.com
  • Export to XML:
    dnsenum --xml output.xml example.com
📘 Features:
  • Discovers subdomains via DNS and Google scraping
  • Performs brute-force using a wordlist
  • Checks for AXFR zone transfers
  • Finds mail (MX), name servers (NS), and host A records
⚠️ Notes:
  • Make sure Perl and required modules are installed (or use preinstalled version in Kali).
  • Zone transfers are rare but highly valuable if successful.
  • Use with caution and only on authorized domains.
SpiderFoot

An automated OSINT tool that scans IPs, domains, emails, and usernames using 200+ sources to reveal vulnerabilities and digital footprints.

OSINT Automation Web/CLI
Run
How to Use SpiderFoot
spiderfoot -s example.com -o simple
🔹 Description:

SpiderFoot is an OSINT automation tool that scans domains, IPs, emails, and usernames to gather intelligence from search engines, breach databases, WHOIS records, darknet, and more. It supports both CLI and web UI modes.

🚀 Common Use Cases:
  • Scan a domain (CLI):
    spiderfoot -s example.com -o simple
  • Launch web UI (on localhost):
    python3 sf.py -l 127.0.0.1:5001
  • Scan IP address:
    spiderfoot -s 192.168.1.1
  • Export results to CSV/JSON:
    spiderfoot -s example.com -o csv -F report.csv
📘 Features:
  • Uses 200+ modules: DNS, WHOIS, Shodan, HaveIBeenPwned, VirusTotal, etc.
  • Supports CLI automation and web interface
  • Visual reporting via charts and graphs
  • Detects subdomains, credentials, breached data, and open ports
⚠️ Notes:
  • Some modules require API keys (e.g., Shodan, Censys, Pastebin).
  • Install with: pip install spiderfoot or clone from GitHub.
  • Use only on targets you’re authorized to scan.
FOCA

A metadata extraction and footprinting tool that analyzes public documents to discover hidden information like usernames, emails, software versions, and network paths.

OSINT Metadata Analysis Windows GUI
Run
How to Use FOCA
🔹 Description:

FOCA is a Windows GUI tool developed by ElevenPaths to automate the process of discovering metadata in documents like PDFs, DOCs, and PPTs. It retrieves files from target domains, extracts hidden metadata, and maps internal systems, usernames, emails, and network paths.

🚀 Common Use Workflow:
  1. Launch FOCA on Windows.
  2. Create a new project and enter the target domain (e.g., example.com).
  3. FOCA will search for publicly available documents via Google, Bing, and DuckDuckGo.
  4. Click "Download" to retrieve all documents.
  5. Use the “Analyze” function to extract metadata like:
    • Author names / usernames
    • Software used
    • Creation/modification timestamps
    • Internal IPs or shared paths
📘 Supported File Types:
  • PDF, DOC, DOCX, XLS, XLSX, PPT, PPTX
  • Extracts metadata such as Author, Company, Software, Comments, Path, and Timestamps
⚠️ Notes:
  • Only available for Windows (GUI-based)
  • Requires internet access to search and download documents
  • Use only on authorized domains or for research purposes
  • Highly useful for social engineering and internal network mapping
📎 Download:

FOCA is available on the official ElevenPaths website:
https://www.elevenpaths.com/labstools/foca.html

Fierce

A DNS reconnaissance tool used to locate non-contiguous IP blocks and discover hidden domains and subnets via brute-force and zone transfers.

Recon DNS Enumeration Zone Transfer
Run
How to Use Fierce
fierce -dns example.com
🔹 Description:

Fierce is a DNS reconnaissance tool that helps find hosts, domains, and IP ranges associated with a target. It performs DNS brute-forcing, reverse lookups, and zone transfers where possible.

🚀 Common Use Cases:
  • Standard DNS recon:
    fierce -dns example.com
  • Use a custom wordlist:
    fierce -dns example.com -wordlist /path/to/list.txt
  • Attempt zone transfer:
    fierce -dns example.com -traverse 5
📘 Explanation of Flags:
  • -dns — The target domain
  • -wordlist — Provide a custom DNS name wordlist
  • -range — IP range to scan (e.g., 192.168.0.0/24)
  • -file — Save output to a file
  • -traverse — Attempts to brute additional DNS servers (for zone transfer testing)
⚠️ Notes:
  • Fierce is focused on DNS, not port scanning or full OSINT.
  • It’s often useful when other DNS tools don’t return much.
  • Use ethically and only on targets you're authorized to test.
NSLookup / Dig

Essential DNS interrogation tools for querying name servers, retrieving zone records, and performing advanced DNS lookups.

DNS Recon Record Lookup CLI Tools
Run
How to Use NSLookup / Dig
dig example.com ANY
                                nslookup example.com
🔹 Description:

NSLookup and Dig are core DNS diagnostic tools used to retrieve DNS records, perform reverse lookups, identify misconfigurations, and assess a domain’s exposure.

🚀 Common Use Cases:
  • Query all DNS records:
    dig example.com ANY
  • Retrieve MX (mail server) records:
    dig example.com MX
  • Get authoritative name servers:
    nslookup -type=ns example.com
  • Check a specific DNS server:
    nslookup example.com 8.8.8.8
  • Reverse lookup:
    dig -x 192.168.1.1
📘 Explanation of Flags:
  • ANY — Requests all available DNS records
  • MX, A, NS, TXT — Specific record types
  • -x — Reverse lookup for IP address
  • -type= — NSLookup-specific for targeting record types
⚠️ Notes:
  • Use dig for more detailed DNS queries and scripting.
  • nslookup is simpler but widely available even on Windows.
  • Both are read-only, safe to use in public recon.

Amass

An advanced OSINT and attack surface mapping tool for discovering subdomains, DNS records, infrastructure, and external assets.

DNS Recon Attack Surface Open Source
Run
How to Use Amass
amass enum -d example.com
🔹 Description:

Amass is a powerful reconnaissance framework that gathers information about a domain’s external attack surface. It performs DNS enumeration, WHOIS analysis, subdomain discovery, IP mapping, and infrastructure correlation using OSINT and active techniques.

🚀 Common Use Cases:
  • Passive subdomain enumeration:
    amass enum -passive -d example.com
  • Active subdomain probing:
    amass enum -active -d example.com
  • Save results to file:
    amass enum -d example.com -o results.txt
  • Visualize network relationships:
    amass viz -d3 -f results.txt -o graph.html
📘 Features:
  • Supports passive and active recon modes
  • Integrates with APIs like VirusTotal, Shodan, Censys, etc.
  • Finds subdomains, IPs, ASN, CIDR blocks, and relationships
  • Can create network graphs for visualization
⚠️ Notes:
  • Install via sudo snap install amass or from GitHub
  • Some modules require free API keys for better results
  • Use amass intel for organization-wide recon
  • Best used with permissions in bug bounty or red team engagements
Censys CLI

A command-line tool to search the Censys internet-wide scan database, uncovering exposed services, misconfigurations, and digital footprints.

Reconnaissance Internet Assets API-Based
Run
How to Use Censys CLI
censys search "services.service_name:HTTP AND location.country:US"
🔹 Description:

Censys CLI is a terminal-based interface to the Censys search engine, which scans and indexes all publicly reachable devices and services on the Internet. It’s used for threat hunting, attack surface mapping, and identifying exposed services or vulnerabilities.

🚀 Common Use Cases:
  • Search for HTTP services in the US:
    censys search "services.service_name:HTTP AND location.country:US"
  • Search for specific software versions:
    censys search "services.software.product:Apache AND services.software.version:2.4.49"
  • Export results to JSON:
    censys search "ip:8.8.8.8" --format json
  • Get full info on a host:
    censys view a.b.c.d
📘 Setup Steps:
  • Install via pip:
    pip install censys
  • Configure with your API key:
    censys config
  • Get API keys from: Censys API Portal
⚠️ Notes:
  • Requires a free or paid Censys account and API credentials.
  • Great for correlating with Shodan or Amass for cross-validation.
  • Supports advanced queries with filters, aggregation, and sorting.
WhatWeb

A web scanner that identifies websites’ technologies, frameworks, server info, CMS, and plugins to reveal potential weaknesses.

Web Recon Technology Fingerprinting Open Source
Run
How to Use WhatWeb
whatweb example.com
🔹 Description:

WhatWeb is a command-line tool that performs web application fingerprinting. It detects technologies used on websites such as web servers, CMSs (e.g., WordPress, Joomla), analytics platforms, JavaScript libraries, security mechanisms, and more.

🚀 Common Use Cases:
  • Basic scan of a site:
    whatweb example.com
  • Enable aggressive detection:
    whatweb -a 3 example.com
  • Scan multiple targets from a file:
    whatweb -i domains.txt
  • Export results to JSON:
    whatweb -a 3 -t 10 -v --log-json=output.json example.com
📘 Features:
  • Detects web technologies and platforms (CMS, frameworks, etc.)
  • Supports multiple detection levels (-a 1 to -a 3)
  • Plugins for fingerprinting: Apache, Google Analytics, jQuery, etc.
  • Flexible output: JSON, verbose, color, logs
⚠️ Notes:
  • Works best with internet-facing websites
  • Be cautious with aggressive scans on production sites
  • Use sudo apt install whatweb to install (or via GitHub)
